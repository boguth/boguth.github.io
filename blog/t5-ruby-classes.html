April 15th, 2015. Let's talk about classes.

Classes are collections of methods that ultimately define the behavior and state of an object. In order to make a class, there must usually be two things: Methods that define what an object of that class can do, and methods to set or retrieve the state of the object.

To get a clearer picture of classes and how they work, let's pick something we would want to represent digitally. How about a pie? We write class named with capitalized first letters and define with class and end.

class Pie
end

There's our pie class. As I mentioned, we must make some methods that define what a pie can do, and some methods that set or retrieve a pie's state.

What are some things a pie can do?

  • It can be eaten.
  • It can be sat on.
  • It can be placed upon a window sill.
Ok cool. Now, what are some states it can have?
  • Baked.
  • Tasty.

Nice. Let's put all that in the class.

class Pie
def baked=(answer)
puts "Is the pie baked? #{answer}."
end

def tasty=(answer)
puts "Is the pie tasty?" #{answer}."
end

def eaten
puts "Your pie has been eaten."
end

def sat
puts "Your pie has been sat on."
end

def fifties_mommed
puts "Your pie has been placed upon a window sill."
end
end
