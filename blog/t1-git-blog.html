Version control is a method of tracking changes to a body of code. This allows coders to both keep up with the changes they've made, as well as revert to older versions of a chunk of code if they are stuck.

Git is a type of version control software that allows for a central, primary repository and as many "branches" off that repository as necessary. From this master repository, a developer can take files off of a larger project, change those files, and then merge them with the master repository.

GitHub provides an online space to store a central repository so that multiple people can work on a project at once. Let's draw an analogy to painting. Let's say three artists are working on one painting of a person. One artist is responsible for painting the head, one for the body, and one for the clothes.

Now, "traditionally," these artists would have to work from the same painting, passing around the same piece of canvas and adding to it. If, however, these artists had a GitHub analog, they would each be able to download a copy of that painting to their own personal canvas, add to it, and then merge that painting with the primary canvas. This lets them work all at once. This is the primary advantage of GitHub.
